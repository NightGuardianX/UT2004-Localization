# PROMPT: Инициализация пайплайна локализации

> AI-промпт. Используется в **целевом** репозитории проекта локализации (отдельном от игрового).

---

## Шаблон промпта

```
Ты инициализируешь структуру документации и пайплайна локализации.
Результат: отдельный репозиторий с локалями и документацией для них как отдельными категориями.

=== ВОПРОСЫ ПЕРЕД НАЧАЛОМ (ответить из конфига или запросить у пользователя) ===

1. **Интеграция с движком**: Локализация встроена в движок (UE, Unity и т.д.) или внешняя через текстовые файлы?
   Варианты: встроенная / внешняя / гибрид. Уточнить формат при внешней.

2. **Инструменты**: Какие инструменты используются? (Crowdin, Lokalise, POedit, кастомные скрипты, AI-агент и т.д.)

3. **Формат файлов**: Стандартный (PO, CSV, JSON) или специфичный для движка (напр. UE2)?
   По умолчанию — PO. Уточнить при инициализации.

4. **Языки и неймспейсы**: Список целевых языков; структура неймспейсов/сущностей (entity).

=== ВХОДНЫЕ ДАННЫЕ ===
Конфиг: см. ЛОК_КОНФИГ_ПРИМЕР.md

=== ЧТО СДЕЛАТЬ ===
1. Прочитай ЛОК_ЧЕКЛИСТ.md и следуй шагам
2. Прочитай ОБЩ_ПРИНЦИПЫ_ДОКУМЕНТАЦИИ.md из _общее/
3. Создай структуру: локали (отдельная категория) + документация для них (отдельная категория)
4. Создай глоссарий локализации и правила валидации
5. Создай документ процесса: анализ → глоссарий → валидация → перевод → валидация → ревью → интеграция

=== ПРАВИЛА ===
- Не придумывай сущности, которых нет в конфиге
- Неизвестное — оставляй пустым или с пометкой TODO
- Имена файлов и папок — подчёркивания, без пробелов
- Ссылки — только на существующие файлы
```

---

## Использование

1. Заполнить `ЛОК_КОНФИГ_ПРИМЕР.md` (ответить на вопросы интеграции, инструментов, формата).
2. Скопировать этот файл, `ЛОК_ЧЕКЛИСТ.md`, `ЛОК_КОНФИГ_ПРИМЕР.md` и `_общее/` в целевой репозиторий локализации.
3. Запустить AI с этим промптом.
4. При необходимости — выполнять чеклист пошагово.
