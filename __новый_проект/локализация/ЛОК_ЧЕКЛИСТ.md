# Чеклист: Инициализация пайплайна локализации

> Пошаговый план. Репозиторий отдельный от игрового; локали и документация — отдельные категории.

## Подготовка

- [ ] Ответить на вопросы из `ЛОК_PROMPT_ИНИЦИАЛИЗАЦИЯ.md` (движок, инструменты, формат)
- [ ] Заполнить `ЛОК_КОНФИГ_ПРИМЕР.md`

## 1. Структура репозитория

- [ ] `локали/` — файлы переводов (PO, CSV и т.д.) по языкам
- [ ] `документация/` или `_документация/` — правила, глоссарии, процесс
- [ ] `глоссарии/` — глоссарий локализации (термины, контексты)
- [ ] `_ai/` (опц.) — промпты для AI-агента перевода

## 2. Документация процесса

- [ ] Описать цикл: **анализ существующего** → **обновление/коррекция глоссария** → **валидация через глоссарий** → **перевод** (ручной / итерация через AI) → **валидация перевода и глоссария** → **ревью** → **интеграция**
- [ ] Правила валидации (что проверять по глоссарию)

## 3. Глоссарий

- [ ] `глоссарии/ГЛОССАРИЙ_ЛОКАЛИЗАЦИИ.md` или аналог
- [ ] Индекс: `глоссарии/_СПИСОК_ГЛОССАРИЕВ.md`
- [ ] Шаблон термина (опц.)

## 4. Правила

- [ ] `Documentation/ПРАВИЛА_ОБЩИЕ.md` — адаптировать из ОБЩ_ПРИНЦИПЫ_ДОКУМЕНТАЦИИ
- [ ] `ПРАВИЛА_ГЛОССАРИИ.md` (или раздел в общих)
- [ ] Правила неймспейсов/сущностей (entity) — по конфигу

## 5. Индексы

- [ ] Список языков
- [ ] Список неймспейсов/сущностей (если применимо)
- [ ] Карта проекта (`AUTO.md` или аналог)

## Связь с общими принципами

- `_общее/ОБЩ_ПРИНЦИПЫ_ДОКУМЕНТАЦИИ.md` — карточки, ссылки, глоссарии, индексы
