# Пайплайны инициализации

> Папка **вне** рабочей документации. Двойное подчёркивание `__` — визуально выше служебных папок `_`.  
> Агент не обращается к этой папке, пока пользователь явно не укажет иное (см. `.cursor/rules/`).

## Назначение

Инструменты для формирования схожей структуры документации в **другом проекте** (игра, пайплайн локализации и т.д.).  
Общие принципы: подсистемы, дата-ассеты, глоссарии, правила оформления — переносятся и адаптируются.

## Нейминг файлов

Префикс показывает **тип пайплайна** и **роль файла**:

| Префикс | Тип пайплайна | Примеры |
|---------|---------------|---------|
| `ИГР_` | Игровой проект (UE, подсистемы, DA, механики) | `ИГР_PROMPT_ИНИЦИАЛИЗАЦИЯ.md` |
| `ЛОК_` | Локализация (будущий пайплайн) | `ЛОК_PROMPT_*.md` |
| `ОБЩ_` | Общие принципы (документы, глоссарии) | `ОБЩ_ПРИНЦИПЫ_ДОКУМЕНТАЦИИ.md` |

| Суффикс/роль | Назначение |
|--------------|------------|
| `PROMPT_` | AI-промпт для инициализации или генерации |
| `ЧЕКЛИСТ_` | Пошаговый план действий |
| `КОНФИГ_` | Пример конфигурации (имя проекта, сущности и т.д.) |
| `ПРИНЦИПЫ_` | Общие подходы к документации, глоссариям |

## Структура

```
__новый_проект/
├── _README.md                 ← этот файл
├── _общее/                    ← принципы, общие для всех пайплайнов
│   └── ОБЩ_ПРИНЦИПЫ_*.md
├── игровой_проект/            ← пайплайн документации игрового проекта
│   ├── ИГР_PROMPT_ИНИЦИАЛИЗАЦИЯ.md
│   ├── ИГР_ЧЕКЛИСТ.md
│   └── ИГР_КОНФИГ_ПРИМЕР.md
└── локализация/               ← пайплайн локализации
    ├── ЛОК_README.md
    ├── ЛОК_PROMPT_ИНИЦИАЛИЗАЦИЯ.md
    ├── ЛОК_ЧЕКЛИСТ.md
    └── ЛОК_КОНФИГ_ПРИМЕР.md
```

## Дальнейшие шаги (игровой проект)

1. Заполнить `ИГР_КОНФИГ_ПРИМЕР.md` под целевой проект.
2. Скопировать `игровой_проект/` и `_общее/` в репозиторий нового проекта.
3. Запустить AI с промптом из `ИГР_PROMPT_ИНИЦИАЛИЗАЦИЯ.md` (подставить данные из конфига) или выполнять `ИГР_ЧЕКЛИСТ.md` пошагово.

Если новый проект пока не планируется — оставить как есть; при необходимости дополнять чеклист и конфиг.

## Связь с рабочими папками

- `_общее/` ссылается на принципы из `Documentation/GENERAL_RULES.md` и правил в `Documentation/`.
- `игровой_проект/` опирается на `_подсистемы/`, `_данные/`, шаблоны — как на эталон.
- При копировании в новый проект — пути и имена адаптируются под целевой репозиторий.
