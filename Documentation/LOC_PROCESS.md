# Процесс локализации (русская локаль)

> Цикл от анализа до интеграции. Источники: `/int`, `/est`. Целевая локаль: `/rut`.

## Этапы

1. **Анализ существующего** — обзор исходных .int/.est, определение файлов без .rut или с неполным переводом. Используется [FILE_TRANSLATION_STATUS](FILE_TRANSLATION_STATUS.md).
2. **Обновление глоссария** — при появлении новых сущностей или терминов: правка глоссариев в [Glossary/](Glossary/) по [GLOSSARY_RULES](GLOSSARY_RULES.md), обновление [_GLOSSARIES_LIST](_GLOSSARIES_LIST.md).
3. **Валидация по глоссарию** — проверка, что переводимые строки учтены в глоссариях сущностей; единообразие терминов (см. [правила валидации](#правила-валидации)).
4. **Перевод** — ручное редактирование .rut; при необходимости итерации с AI-агентом с опорой на глоссарии.
5. **Валидация перевода** — проверка по [FILE_TRANSLATION_STATUS_RULES](FILE_TRANSLATION_STATUS_RULES.md): статус (Untranslated, In Progress, Done и т.д.), наличие `; EN:` для переводимых ключей, отсутствие испанских плейсхолдеров.
6. **Ревью** — ручная вычитка и фиксация в таблице статуса (Verified).
7. **Интеграция** — копирование .rut в установку игры.

## Правила валидации

- **По глоссарию:** термины из глоссариев сущностей (Game_*, UI_*, Lore_*, Tech_*) должны использоваться единообразно в .rut в рамках одной сущности.
- **По статусу файла:** для каждого .rut определяется Base File (.int или .est) и Status по [FILE_TRANSLATION_STATUS_RULES](FILE_TRANSLATION_STATUS_RULES.md); счётчики на странице статуса обновляются при изменениях.
- **Формат .rut:** переводимые строки в виде `; EN: Key="English"` + `Key="Russian"`; нет переводимых ключей без `; EN:` или без русского значения.
