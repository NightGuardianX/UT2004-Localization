# Процесс локализации (русская локаль)

> Цикл от анализа до интеграции. Источники: `/int`, `/est`. Целевая локаль: `/rut`.

## Этапы

1. **Анализ существующего** — обзор исходных .int/.est, определение файлов без .rut или с неполным переводом. Используется [File Translation Status](File%20Translation%20Status.md).
2. **Обновление глоссария** — при появлении новых сущностей или терминов: правка глоссариев в [Glossary/](Glossary/) по [Glossary Rules](Glossary%20Rules.md), обновление [_СПИСОК_ГЛОССАРИЕВ](_СПИСОК_ГЛОССАРИЕВ.md).
3. **Валидация по глоссарию** — проверка, что переводимые строки учтены в глоссариях сущностей; единообразие терминов (см. [правила валидации](#правила-валидации)).
4. **Перевод** — ручное редактирование .rut; при необходимости итерации с AI-агентом с опорой на глоссарии.
5. **Валидация перевода** — проверка по [File Translation Status Rules](File%20Translation%20Status%20Rules.md): статус (Untranslated, In Progress, Done и т.д.), наличие `; EN:` для переводимых ключей, отсутствие испанских плейсхолдеров.
6. **Ревью** — ручная вычитка и фиксация в таблице статуса (Verified).
7. **Интеграция** — копирование .rut в установку игры.

## Правила валидации

- **По глоссарию:** термины из глоссариев сущностей (Game_*, UI_*, Lore_*, Tech_*) должны использоваться единообразно в .rut в рамках одной сущности.
- **По статусу файла:** для каждого .rut определяется Base File (.int или .est) и Status по [File Translation Status Rules](File%20Translation%20Status%20Rules.md); счётчики на странице статуса обновляются при изменениях.
- **Формат .rut:** переводимые строки в виде `; EN: Key="English"` + `Key="Russian"`; нет переводимых ключей без `; EN:` или без русского значения.
